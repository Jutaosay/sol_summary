# 三 Agent 分工 / 验收标准 / 回归清单

> 目标：在最短时间内完成 `sol_summary` 的可运行交付（代码 + 文档 + 运维配置），并降低上线前风险。

## 1) 三 Agent 分工

### Agent-1（Core / Backend）
**职责**
- 核心业务逻辑实现：采集、统计、持久化主流程
- API 行为与参数校验
- 异常处理与日志结构（至少可定位 rpc/计算/存储三类问题）

**交付物**
- `src/sol_summary/*` 核心代码
- 关键接口与数据模型
- 至少一条端到端跑通链路（track -> metric 入库 -> 查询）

---

### Agent-2（QA / Validation）
**职责**
- 用例设计与执行（单测、冒烟、关键路径手测）
- 输出缺陷清单（严重级别、复现步骤、建议）
- 验证修复后的回归结果

**交付物**
- 测试结果（通过/失败、覆盖范围、风险点）
- 缺陷单或 issue 列表
- 回归确认结论（是否达到可发布标准）

---

### Agent-3（Docs / Ops）
**职责**
- 文档完善（README、操作说明、故障处理）
- 环境变量模板与部署配置建议
- 研发/测试/运维协作信息沉淀（分工、验收、回归 checklist）

**交付物**
- README：架构、API、429 排障、自定义 RPC
- `.env.example`：关键配置项模板
- `docs/task_split.md`：团队协作基线文档

---

## 2) 验收标准（Definition of Done）

### A. 功能验收
1. 能成功启动服务并打开 Web 页面。
2. `POST /track` 后可在 DB 中看到持续新增 metric。
3. `GET /latest`、`GET /metrics` 返回结构正确且有数据。
4. `POST /stop` 后目标 CA 不再新增采样点。

### B. 数据验收
1. 指标字段完整（holder_count/top50/trimmed 等）。
2. 时间序列按时间递增可用于前端绘图。
3. 发生异常时有 `job_log` 记录（status=error + err_msg）。

### C. 文档验收
1. README 覆盖架构、API、环境变量、排障。
2. 包含 RPC 429 明确处置方案。
3. 提供自定义 RPC 配置方法。
4. `.env.example` 可直接复制后使用。

### D. 运维验收
1. 刷新间隔、RPC、DB 路径可通过环境变量修改。
2. 默认值合理（不配置也能启动）。
3. 关键风险（限流、超时、数据近似口径）有明确提示。

## 3) 回归清单（Release Regression Checklist）

## 3.1 启动与配置
- [ ] 使用默认配置启动成功
- [ ] 使用自定义 `SOL_RPC_URL` 启动成功
- [ ] 使用自定义 `SOL_SUMMARY_DB` 可正常写入
- [ ] 调整 `SOL_SUMMARY_REFRESH_SECONDS` 后采样频率生效

## 3.2 API 行为
- [ ] `/track` 重复提交同一 CA 行为符合预期（幂等或明确返回）
- [ ] `/stop` 对不存在 CA 返回符合预期
- [ ] `/latest` 在无数据时返回 404
- [ ] `/metrics` limit 边界值（1、120、5000）行为正确

## 3.3 数据链路
- [ ] 追踪后 `holder_metrics` 持续新增
- [ ] `job_log` 在成功/失败场景均有记录
- [ ] `trimmed_count` 与 top50 切片逻辑一致
- [ ] 前端图表可随数据刷新

## 3.4 异常与稳健性
- [ ] RPC 429 场景下系统可恢复（重试后继续采样）
- [ ] 临时网络超时不会导致进程退出
- [ ] 非法 CA 输入不导致服务崩溃
- [ ] 高并发多 CA 下资源占用可接受

## 3.5 文档一致性
- [ ] README 的 API 与实际路由一致
- [ ] README 的环境变量名与代码一致
- [ ] README 中端口与启动命令一致
- [ ] `.env.example` 项目可读、注释清晰

---

## 4) 交接建议

- 代码变更后，先执行最小回归（启动 + 单 CA track + latest/metrics）。
- 上线前至少做一次“公共 RPC 压测 + 私有 RPC 验证”对比，确认限流风险。
- 若未来引入全量 holder 口径，需同步更新 README 的“数据口径说明”。
